## 需求 1：支持起拍价出价

### 问题
Yahoo 商品有些是比较高的价格起拍的，在没有其他用户出价的时候，起拍价也可以出价。但之前的校验要求最高价必须 > 当前价，导致无法以起拍价出价。

### 解决方案
修改提交校验逻辑：最高价的税前价 **>=** 商品目前的税前价（修改前要求 **>**）

### 测试结果
✓ 8/8 通过
- 普通商品起拍价出价
- 商城商品起拍价出价（含税计算）
- 边界情况验证

### 影响文件
- `src/client/src/pages/Submit.jsx`

## 需求 2：后台订单管理字段完善

### 问题
后台订单管理页面缺少用户名、运费、银行手续费等关键字段，应付款计算公式不完整。

### 解决方案

#### 新增字段
1. **用户名**：显示下单用户
2. **运费**：显示 4 种形式（無料、着払い、落札者負担、固定金额）
3. **银行手续费(日元)**：新增配置参数，默认 0
4. **手续费(RMB)**：原"手续费"改为 RMB 单位

#### 应付款公式更新
```
修改前：(落札金额 + 手续费) * 汇率
修改后：(落札金额 + 运费 + 银行手续费) * 汇率 + 手续费(RMB)
```

#### 示例计算
```
落札金额：20,000円
运费：1,000円
银行手续费：500円
汇率：0.05
手续费(RMB)：15元
应付款 = (20000 + 1000 + 500) * 0.05 + 15 = 1090元
```

### 测试结果
- ✓ 运费解析 9/9 通过
- ✓ 应付款计算 4/4 通过

### 影响文件
- `src/admin/src/Orders.tsx`
- `src/server/routes/admin.js`

## 需求 3：后台菜单折叠功能（移动端优化）

### 问题
后台页面需要在手机上查看，但菜单太大，手机看不到右边的页面内容。

### 解决方案

#### 第一阶段：基础折叠功能
- 参考 YouTube 样式，左上角增加三横线图标按钮
- 点击可展开/收起侧边栏
- 折叠时隐藏 Yahoo 登录状态提示

#### 第二阶段：移动端优化
- 折叠宽度从 80px 减少到 **50px**（-38%）
- 折叠时只显示菜单第一个字
- 固定定位，滚动时始终可见
- 平滑过渡动画（0.2s）

#### 第三阶段：去除滚动条
- 侧边栏 `overflow: auto` 改为 `overflow: hidden`
- 展开状态无滚动条，界面更简洁

## 注意事项

### 起拍价出价
- 现在支持以起拍价出价
- 用户可以在无人竞价时直接出起拍价
- 仍然不允许低于当前价出价

- 运费只在商品提交时解析
- 落札同步不更新运费

- 新公式包含运费和银行手续费
- 请在后台配置正确的参数
- 配置会保存到数据库 config 表

- 折叠状态不会持久化
- 刷新页面后恢复展开状态
- 建议移动端使用折叠状态
