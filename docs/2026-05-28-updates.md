# 2026-05-28 更新说明

## 更新内容

### 1. 支持起拍价出价

**问题**：Yahoo 商品有些是比较高的价格起拍的，在没有其他用户出价的时候，起拍价也可以出价。但之前的校验要求最高价必须 > 当前价，导致无法以起拍价出价。

**修改**：
- 修改提交校验逻辑：最高价的税前价 >= 商品目前的税前价即可提交
- 修改前：`submitMaxPrice > currentPrice`（不支持起拍价）
- 修改后：`submitMaxPrice >= currentPrice`（支持起拍价）

**影响文件**：
- `src/client/src/pages/Submit.jsx`

**测试**：
- ✓ 普通商品起拍价出价（等于当前价）
- ✓ 商城商品起拍价出价（税后等于当前价）
- ✓ 高于起拍价出价
- ✓ 低于当前价拒绝

---

### 2. 后台订单管理字段完善

**问题**：后台订单管理页面缺少用户名、运费、银行手续费等关键字段，应付款计算公式不完整。

**修改**：

#### 2.1 新增字段
- **用户名**：显示下单用户
- **运费**：显示 4 种形式（無料、着払い、落札者負担、固定金额）
- **银行手续费(日元)**：新增配置参数，默认 0
- **手续费(RMB)**：原"手续费"改为 RMB 单位

#### 2.2 应付款公式更新
```
修改前：应付款 = (落札金额 + 手续费) * 汇率
修改后：应付款 = (落札金额 + 运费 + 银行手续费) * 汇率 + 手续费(RMB)
```

#### 2.3 运费解析
支持以下格式：
- `送料 無料` → 0円
- `送料 着払い` → 0円
- `送料 落札者負担` → 0円
- `送料 1,000円` → 1000円
- `送料 500円` → 500円

**影响文件**：
- `src/admin/src/Orders.tsx`
- `src/server/routes/admin.js`

**测试**：
- ✓ 运费解析（9种格式）
- ✓ 应付款计算（4种场景）
- ✓ 公式验证

---

### 3. 后台菜单折叠功能

**问题**：后台菜单始终展开，占用屏幕空间。

**修改**：
- 参考 YouTube 样式，增加菜单折叠功能
- 左上角增加三横线图标按钮
- 点击可展开/收起侧边栏
- 折叠时隐藏 Yahoo 登录状态提示

**影响文件**：
- `src/admin/src/layouts/AdminLayout.tsx`

**功能**：
- ✓ 点击按钮切换折叠状态
- ✓ 折叠时显示图标菜单
- ✓ 展开时显示完整菜单和状态

## 注意事项

1. **起拍价出价**：现在支持以起拍价出价，用户可以在无人竞价时直接出起拍价
2. **运费计算**：运费只在商品提交时解析，落札同步不更新运费
3. **应付款公式**：新公式包含运费和银行手续费，请在后台配置正确的参数
4. **菜单折叠**：折叠状态不会持久化，刷新页面后恢复展开状态
